## 4. Distribution timing — why the order matters

The three external launches (Show HN, X thread, LinkedIn) must not go out before `@arbitova/sdk@3.0.0` and `arbitova@2.5.0` are live on their respective registries. HN readers will `npm install @arbitova/sdk` and `pip install 'arbitova[path_b]'` within three minutes of the Show HN post landing on the front page. If those installs fail, the thread dies.

**Recommended order:**
1. User pastes PyPI token → I run `twine upload`.
2. User pastes npm token → I run `npm publish @arbitova/sdk@3.0.0` (and deprecate 2.3.1).
3. User pastes Render admin key → I run `publish.sh dev-log-015`.
4. Verify all three install/fetch clean.
5. Post X thread + LinkedIn simultaneously (Tue–Thu morning PT).
6. ~2 hours later, if no regressions, Show HN.

Steps 1–3 are ~10 minutes of token-paste + ~30 seconds of my execution each. Everything else is already queued.
